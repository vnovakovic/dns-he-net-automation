Create `internal/browser/launcher.go`:
- Package `browser`
- Define `Launcher` struct with fields: pw *playwright.Playwright, browser playwright.Browser, headless bool, slowMo float64
- `func NewLauncher(headless bool, slowMo float64) (*Launcher, error)`:
  1. `playwright.Run()` to start the Node.js driver process.
  2. Build `BrowserTypeLaunchOptions{Headless: playwright.Bool(headless)}`. Add `SlowMo: playwright.Float(slowMo)` if slowMo > 0.
  3. `pw.Chromium.Launch(opts)`. If launch fails, call `pw.Stop()` before returning error.
  4. Log with slog.Info: "browser launched", "headless", headless, "slowMo", slowMo.
  5. Return `&Launcher{...}`.
- `func (l *Launcher) NewAccountContext(defaultTimeoutMs float64) (playwright.BrowserContext, error)`:
  1. `l.browser.NewContext()` to create an isolated context.
  2. `ctx.SetDefaultTimeout(defaultTimeoutMs)` to set operation timeout for all actions in this context.
  3. Return the context.
- `func (l *Launcher) Close()`:
  1. If browser != nil, call `l.browser.Close()`.
  2. If pw != nil, call `l.pw.Stop()`.
  3. Log "browser closed".
- `func (l *Launcher) IsConnected() bool`: Return `l.browser.IsConnected()` -- used by session health checks.

Create `internal/browser/session.go`:
- Define `AccountSession` struct:
  - `mu sync.Mutex` -- per-account serialization lock
  - `ctx playwright.BrowserContext` -- isolated browser context
  - `page playwright.Page` -- the active page in this context
  - `accountID string`
  - `lastLogin time.Time`
  - `healthy bool`
- Define `SessionManager` struct:
  - `launcher *Launcher`
  - `credProvider credential.Provider` -- import from internal/credential
  - `sessions map[string]*AccountSession`
  - `sessionsMu sync.RWMutex` -- protects the sessions map (NOT the per-account mutex)
  - `queueTimeout time.Duration` -- from OPERATION_QUEUE_TIMEOUT_SEC config
  - `opTimeout time.Duration` -- from OPERATION_TIMEOUT_SEC config
  - `reloginAge time.Duration` -- from SESSION_MAX_AGE_SEC config
  - `minOpDelay time.Duration` -- from MIN_OPERATION_DELAY_SEC config
- `func NewSessionManager(launcher *Launcher, credProvider credential.Provider, queueTimeout, opTimeout, reloginAge, minOpDelay time.Duration) *SessionManager`
- `func (sm *SessionManager) getOrCreateSession(accountID string) *AccountSession`:
  1. Read-lock sessionsMu, check if session exists.
  2. If not, write-lock, double-check, create new AccountSession with accountID set, healthy=false.
  3. Return session.
- `func (sm *SessionManager) WithAccount(ctx context.Context, accountID string, op func(playwright.Page) error) error`:
  1. Get or create session via getOrCreateSession.
  2. Queue with timeout: spawn goroutine that locks session.mu and closes a channel. Select on: channel (acquired), time.After(queueTimeout) (return ErrQueueTimeout), ctx.Done() (return ctx.Err()). IMPORTANT: if timeout/cancel fires but the goroutine eventually acquires the lock, it must unlock. Use a done channel + select in the goroutine to handle this race: the goroutine should check after acquiring the lock whether the caller already timed out, and if so, unlock and return.
  3. Wrap operation in opTimeout: `opCtx, cancel := context.WithTimeout(ctx, sm.opTimeout)`, defer cancel.
  4. Call `sm.ensureHealthy(opCtx, session)` -- if error, return wrapped ErrSessionUnhealthy.
  5. Apply minimum operation delay: if time since last operation < minOpDelay, sleep the difference. This prevents hammering dns.he.net.
  6. Call `op(session.page)`.
  7. Return result.
- `func (sm *SessionManager) ensureHealthy(ctx context.Context, session *AccountSession) error`:
  - For now (Plan 01-02), this is a STUB that:
    1. If session.ctx == nil or !session.healthy: create new context via launcher.NewAccountContext, create new page, set session.ctx/page/healthy.
    2. If session healthy: return nil.
  - The actual health check logic (login verification, re-login) is added in Plan 01-03 when page objects exist.
  - Log decisions: "creating new session" or "session healthy".
- `func (sm *SessionManager) Close()`:
  1. Lock sessionsMu.
  2. For each session: close session.ctx (BrowserContext.Close()).
  3. Clear sessions map.
  4. Log "all sessions closed".
