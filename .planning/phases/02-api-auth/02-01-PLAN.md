Create `internal/store/migrations/002_tokens.sql`:

```sql
-- +goose Up
CREATE TABLE IF NOT EXISTS tokens (
    jti        TEXT PRIMARY KEY,
    account_id TEXT NOT NULL
               REFERENCES accounts(id)
               ON DELETE CASCADE,
    role       TEXT NOT NULL CHECK (role IN ('admin', 'viewer')),
    label      TEXT,
    token_hash TEXT NOT NULL UNIQUE,
    expires_at DATETIME,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    revoked_at DATETIME
);

CREATE INDEX IF NOT EXISTS idx_tokens_account_id ON tokens(account_id);

-- +goose Down
DROP INDEX IF EXISTS idx_tokens_account_id;
DROP TABLE IF EXISTS tokens;
```

Schema design notes:
- `jti` as PRIMARY KEY: O(log n) revocation lookup with no additional index.
- `token_hash UNIQUE`: prevents duplicate token registrations.
- `ON DELETE CASCADE`: deleting an account removes all its tokens (ACCT-03).
- `expires_at` nullable: NULL means no expiry (TOKEN-04).
- `revoked_at` nullable: NULL means active (TOKEN-05).
- `role CHECK`: DB-level enforcement against invalid role strings.
- Do NOT add a `password` or credential column — credentials stay in Vault/env (SEC-01).

Create `internal/token/token.go` with package `token`. This file contains all token business logic. No HTTP code belongs here.

**Claims struct:**

```go
import "github.com/golang-jwt/jwt/v5"

// Claims is the JWT payload for dns-he-net-automation bearer tokens.
// Embeds jwt.RegisteredClaims which provides: ID (jti), ExpiresAt, IssuedAt.
type Claims struct {
    AccountID string `json:"account_id"`
    Role      string `json:"role"` // "admin" or "viewer"
    Label     string `json:"label,omitempty"`
    jwt.RegisteredClaims
}
```

**TokenRecord struct** (used by ListTokens — never exposes token_hash or raw token):

```go
type TokenRecord struct {
    JTI       string     `json:"jti"`
    AccountID string     `json:"account_id"`
    Role      string     `json:"role"`
    Label     *string    `json:"label,omitempty"`
    CreatedAt time.Time  `json:"created_at"`
    ExpiresAt *time.Time `json:"expires_at,omitempty"`
    RevokedAt *time.Time `json:"revoked_at,omitempty"`
}
```

**IssueToken function:**

```go
func IssueToken(ctx context.Context, db *sql.DB, accountID, role, label string, expiresInDays int, secret []byte) (rawToken string, jti string, err error)
```

Implementation steps:
1. Validate role is "admin" or "viewer"; return error otherwise.
2. Compute `expiresAt *time.Time`: if expiresInDays > 0, set to time.Now().AddDate(0, 0, expiresInDays); if 0, leave nil (unlimited, TOKEN-04).
3. Generate jti via `uuid.New().String()` (github.com/google/uuid is already in go.mod as indirect).
4. Build Claims with AccountID, Role, Label, and RegisteredClaims{ID: jti, IssuedAt: jwt.NewNumericDate(time.Now())}. Set ExpiresAt only if expiresAt != nil.
5. Sign with `jwt.NewWithClaims(jwt.SigningMethodHS256, claims).SignedString(secret)` — the returned string IS the raw token.
6. Hash: `h := sha256.Sum256([]byte(rawToken)); tokenHash := hex.EncodeToString(h[:])`.
7. Store in DB: `INSERT INTO tokens (jti, account_id, role, label, token_hash, expires_at) VALUES (?, ?, ?, ?, ?, ?)`.
   - For label: pass nil if empty string. For expires_at: pass nil if expiresAt == nil.
8. Return rawToken and jti (no error on success). rawToken is displayed to the user once; it is NEVER stored anywhere — only token_hash is in the DB (TOKEN-02, SEC-02).

**ValidateToken function:**

```go
func ValidateToken(ctx context.Context, db *sql.DB, rawToken string, secret []byte) (*Claims, error)
```

Implementation steps:
1. Parse JWT: `jwt.ParseWithClaims(rawToken, &claims, keyFunc, jwt.WithValidMethods([]string{"HS256"}))`.
   - keyFunc: check `t.Method.(*jwt.SigningMethodHMAC)` — if not HMAC, return error (prevents algorithm confusion attack per research pitfall #1).
   - Return secret as key.
   - CRITICAL: always pass `jwt.WithValidMethods([]string{"HS256"})` — this is the primary defense against algorithm switching attacks.
2. If parse fails or `!token.Valid`, return nil + wrapped error.
3. Revocation check: compute `tokenHash := hex.EncodeToString(sha256.Sum256([]byte(rawToken))[:])`.
   Query: `SELECT revoked_at FROM tokens WHERE jti = ? AND token_hash = ?` with claims.ID and tokenHash.
   - If sql.ErrNoRows: return nil + error "token not found".
   - If revoked_at IS NOT NULL (use sql.NullTime): return nil + error "token revoked".
4. Return &claims, nil on success.

**RevokeToken function:**

```go
func RevokeToken(ctx context.Context, db *sql.DB, accountID, jti string) error
```

Implementation:
1. `UPDATE tokens SET revoked_at = CURRENT_TIMESTAMP WHERE jti = ? AND account_id = ?`.
2. Check `result.RowsAffected()` — if 0 rows updated, return sql.ErrNoRows (caller maps to 404).
3. Return nil on success.

**ListTokens function:**

```go
func ListTokens(ctx context.Context, db *sql.DB, accountID string) ([]TokenRecord, error)
```

Implementation:
1. `SELECT jti, account_id, role, label, created_at, expires_at, revoked_at FROM tokens WHERE account_id = ? ORDER BY created_at DESC`.
2. Scan each row into TokenRecord. Use sql.NullString for label, sql.NullTime for expires_at and revoked_at.
3. Populate *string and *time.Time pointers from Null types: if Valid, set pointer; if not, leave nil.
4. NEVER include token_hash in SELECT or in the returned struct (TOKEN-06, SEC-02).
