The VaultProvider must:

1. **NewVaultProvider(cfg)** â€” constructor that creates a Vault API client and authenticates:

2. **GetCredential(ctx, accountID)** â€” lazy fetch with TTL cache:
   - Step 1: RLock, check cache hit (entry exists AND time.Since(entry.fetchedAt) < ttl), return if hit
   - Step 2: RUnlock, fetch from Vault: `client.KVv2(mountPath).Get(ctx, fmt.Sprintf(pathTmpl, accountID))`
   - Step 3: On fetch error, re-RLock and check stale cache; if stale entry exists, log `slog.WarnContext` "vault unreachable, serving stale credential" and return it; otherwise return the error
   - Step 4: On success, extract `username` and `password` from `secret.Data` via type assertion `.(string)`
   - Step 5: Lock, double-check cache (another goroutine may have populated it), update cache, Unlock
   - SECURITY: NEVER log password or username values. Never put them in error messages.

3. **ListAccountIDs(ctx, accountID)** â€” VaultProvider cannot enumerate accounts (Vault KV list requires different permissions and is not part of the interface contract). Return `[]string{}, nil` as a stub. Add a comment explaining that listing is not supported for VaultProvider; account IDs come from SQLite accounts table.

4. **cachedCred struct** â€” internal type: `{ cred *Credential; fetchedAt time.Time }`

5. **VaultProvider struct** â€” fields: `client *api.Client`, `mountPath string`, `pathTmpl string`, `ttl time.Duration`, `mu sync.RWMutex`, `cache map[string]*cachedCred`
