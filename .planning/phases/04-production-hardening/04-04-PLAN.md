**cmd/server/main.go:**

1. **Credential provider selection** — replace the hardcoded `credential.NewEnvProvider` call with:
```go
var credProvider credential.Provider
if cfg.VaultAddr != "" {
    slog.Info("using Vault credential provider", "vault_addr", cfg.VaultAddr, "auth_method", cfg.VaultAuthMethod)
    vaultProvider, err := credential.NewVaultProvider(cfg)
    if err != nil {
        slog.Error("failed to initialize Vault provider", "error", err)
        os.Exit(1)
    }
    credProvider = vaultProvider
} else {
    slog.Info("using env credential provider (HE_ACCOUNTS)")
    envProvider, err := credential.NewEnvProvider(cfg.HEAccountsJSON)
    if err != nil {
        slog.Error("failed to initialize env credential provider", "error", err)
        os.Exit(1)
    }
    credProvider = envProvider
}
```

Remove the old `ids, err := credProvider.ListAccountIDs(...)` log if VaultProvider is selected (VaultProvider returns empty list). Only log account IDs for EnvProvider, where they're meaningful:

if cfg.VaultAddr == "" {

**internal/api/router.go:**

Update `NewRouter` signature to accept:
```go
func NewRouter(db *sql.DB, sm *browser.SessionManager, launcher *browser.Launcher,
    secret []byte, breakers *resilience.BreakerRegistry,
    globalRPM, perTokenRPM int,
    vaultHealthFn func() string) http.Handler
```

Register middleware in correct order (research Pitfall 4 — global before auth, per-token after auth):
```go
// Global rate limit (before auth — DDoS protection layer)
r.Use(middleware.GlobalRateLimit(globalRPM))

r.Use(chiMiddleware.RequestID)
r.Use(chiMiddleware.RealIP)
// ... panic recovery middleware (unchanged) ...

// /api/v1 routes:
r.Route("/api/v1", func(r chi.Router) {
    r.Use(middleware.BearerAuth(db, secret))
    // Per-token rate limit (after auth — needs token from Authorization header)
    r.Use(middleware.PerTokenRateLimit(perTokenRPM))
    // ... existing routes unchanged ...
})
```

Pass `vaultHealthFn` to `handlers.HealthHandler(db, launcher, vaultHealthFn)`.

Pass `breakers` to handlers that perform browser operations (ListZones, CreateZone, DeleteZone, ListRecords, GetRecord, CreateRecord, UpdateRecord, DeleteRecord). Each of these handlers will wrap their `sm.WithAccount(...)` call with:
```go
err := breakers.Execute(ctx, accountID, func() error {
    return resilience.WithRetry(ctx, func(ctx context.Context) error {
        return sm.WithAccount(ctx, accountID, func(page playwright.Page) error {
            // existing operation
        })
    })
})
```

IMPORTANT: This wrapping pattern should be added to the zone and record handlers. The handler files (zones.go, records.go) need to be updated to use the breaker+retry pattern. Add `breakers *resilience.BreakerRegistry` as a parameter to the zone and record handler constructor functions that make browser calls. Account and token handlers do NOT need retry/breaker (they only touch SQLite).

**internal/api/handlers/health.go:**

Update `HealthHandler` to accept a `vaultHealthFn func() string` parameter:
```go
func HealthHandler(db *sql.DB, launcher *browser.Launcher, vaultHealthFn func() string) http.HandlerFunc
```

In the handler body, add vault status check:
```go
// Vault connectivity status (VAULT-04)
vaultStatus := vaultHealthFn()
checks["vault"] = vaultStatus
if vaultStatus != "ok" && vaultStatus != "disabled" {
    status = "degraded"
}
```

The `vaultHealthFn` is provided by main.go. When VaultProvider is used, it calls `client.Sys().Health()` and returns "ok" or "degraded: <reason>". When EnvProvider is used, it returns "disabled" (no Vault configured). The function is a closure captured at startup.
