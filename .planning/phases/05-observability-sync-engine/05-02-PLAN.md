CRITICAL anti-pattern: Do NOT use `r.URL.Path` as a label — this causes cardinality explosion with zone IDs in the path. Always use `chi.RouteContext(r.Context()).RoutePattern()`.

**Step 2 — Instrument WithAccount():**
In the `WithAccount()` function body, add:

Before acquiring the mutex (queue depth tracking):
```go
if sm.metrics != nil {
    sm.metrics.QueueDepth.WithLabelValues(accountID).Inc()
}
```

After acquiring the mutex (decrement queue depth):
```go
if sm.metrics != nil {
    sm.metrics.QueueDepth.WithLabelValues(accountID).Dec()
}
```

Before calling the user op function, record start time:
```go
start := time.Now()
```

After the op function returns (inside WithAccount, after `err := op(...)`):
```go
if sm.metrics != nil {
    result := "success"
    if err != nil {
        result = "error"
    }
    sm.metrics.BrowserOpsTotal.WithLabelValues(opType, result).Inc()
    sm.metrics.BrowserOpDuration.WithLabelValues(opType).Observe(time.Since(start).Seconds())
}
```

The `opType` parameter: add a new `opType string` parameter to `WithAccount()`. All existing call sites in handlers (zones.go, records.go, health.go, accounts.go) must pass an op type string. Use fine-grained op types matching the operation:
- `"list_zones"`, `"create_zone"`, `"delete_zone"` for zone operations
- `"list_records"`, `"create_record"`, `"update_record"`, `"delete_record"`, `"find_record"` for record operations
- `"health_check"` for health endpoint
- `"login"` for login operations inside ensureHealthy/ForceRelogin

**Step 3 — Instrument session lifecycle (active sessions gauge):**
In the function that creates a new session (after successful login and context creation), add:
```go
if sm.metrics != nil {
    sm.metrics.ActiveSessions.Inc()
}
```

In the function that closes/destroys a browser session (context close), add:
```go
if sm.metrics != nil {
    sm.metrics.ActiveSessions.Dec()
}
```

CRITICAL: All metric calls must be nil-guarded with `if sm.metrics != nil { ... }` — this allows unit tests to pass nil and skip all metric calls without panicking.

CRITICAL: The opType observation call MUST be placed AFTER `op()` returns — measure actual operation time, not setup time.

Update all callers of `NewSessionManager()` to pass the metrics registry. In `cmd/server/main.go`, pass the registry created by `metrics.NewRegistry()`. In test files, pass nil.
