Run `go get github.com/prometheus/client_golang@v1.23.2` to add the dependency.

Create `internal/metrics/metrics.go` with package `metrics`.

Define the `Registry` struct holding:
- `reg *prometheus.Registry` (unexported, used internally for Handler)
- `HTTPRequestsTotal *prometheus.CounterVec` — labels: method, route, status
- `HTTPRequestDuration *prometheus.HistogramVec` — labels: method, route; buckets: []float64{.05, .1, .25, .5, 1, 2.5, 5, 10, 15, 30} (DNS scraping takes 2-10s)
- `BrowserOpsTotal *prometheus.CounterVec` — labels: op_type, result
- `BrowserOpDuration *prometheus.HistogramVec` — labels: op_type; buckets: []float64{.5, 1, 2.5, 5, 10, 15, 30}
- `ActiveSessions prometheus.Gauge` — no labels
- `QueueDepth *prometheus.GaugeVec` — labels: account_id
- `ErrorsTotal *prometheus.CounterVec` — labels: error_type
- `SyncOpsTotal *prometheus.CounterVec` — labels: op_type (add/update/delete), result (ok/error)

All metric names use namespace "dnshe" with appropriate subsystems:
- HTTP layer: subsystem "http" → dnshe_http_requests_total, dnshe_http_request_duration_seconds
- Browser: subsystem "browser" → dnshe_browser_operations_total, dnshe_browser_operation_duration_seconds, dnshe_browser_active_sessions, dnshe_browser_queue_depth
- App errors: subsystem "app" → dnshe_app_errors_total
- Sync: subsystem "sync" → dnshe_sync_operations_total

Implement `NewRegistry() *Registry`:
1. Call `prometheus.NewRegistry()` to get a fresh custom registry (NOT prometheus.DefaultRegisterer — avoid test panics on duplicate registration)
2. Register Go runtime and process collectors: `reg.MustRegister(collectors.NewGoCollector(), collectors.NewProcessCollector(collectors.ProcessCollectorOpts{}))`
3. Create `f := promauto.With(reg)`
4. Use `f.NewCounterVec`, `f.NewHistogramVec`, `f.NewGaugeVec`, `f.NewGauge` to create all metrics with the opts and label names above
5. Return the populated Registry struct

Implement `Handler() http.Handler`:
- Returns `promhttp.HandlerFor(r.reg, promhttp.HandlerOpts{})` for serving /metrics

CRITICAL anti-patterns to avoid (from research):
- Never use `prometheus.DefaultRegisterer` or `promauto.NewCounterVec(...)` without `.With(reg)` — causes panic on duplicate registration in tests
- Never register a metric twice — all registration happens inside NewRegistry()

Imports needed:

- `github.com/prometheus/client_golang/prometheus/collectors`
- `github.com/prometheus/client_golang/prometheus/promauto`
