**Handler factory signature** (match existing handler factories in records.go/zones.go):
```go
func SyncRecords(db *sql.DB, sm *browser.SessionManager, breakers *resilience.BreakerRegistry, reg *metrics.Registry) http.HandlerFunc {
```
The `reg *metrics.Registry` parameter may be nil (nil-guard all metric calls).

**Handler body logic:**

1. Extract claims: `claims := middleware.ClaimsFromContext(r.Context())` — if nil, respond 401 and return
2. Extract zoneID: `zoneID := chi.URLParam(r, "zoneID")`
3. Check dry_run: `dryRun := r.URL.Query().Get("dry_run") == "true"`
4. Decode desired state from request body:
   ```go
   var desiredRecords []model.Record
   if err := json.NewDecoder(r.Body).Decode(&desiredRecords); err != nil {
       response.WriteError(w, http.StatusBadRequest, "invalid_body", "Request body must be a JSON array of records")
       return
   }
   ```
   Handle nil body / empty array gracefully (empty desired = delete all managed records).

5. Scrape live state via browser (pattern from records.go ListRecords handler):
   ```go
   var currentRecords []model.Record
   err := breakers.Execute(r.Context(), claims.AccountID, func() error {
       return resilience.WithRetry(r.Context(), func(ctx context.Context) error {
           return sm.WithAccount(ctx, claims.AccountID, "list_records", func(page playwright.Page) error {
               rp := pages.NewRecordPage(page)
               recs, err := rp.ListRecords(zoneID)
               currentRecords = recs
               return err
           })
       })
   })
   if err != nil {
       handleBrowserError(w, err)
       return
   }
   ```

6. Compute diff: `plan := reconcile.DiffRecords(currentRecords, desiredRecords)`

7. If dryRun, respond immediately:
   ```go
   response.WriteJSON(w, http.StatusOK, reconcile.SyncResponse{DryRun: true, Plan: plan})
   return
   ```

8. Apply changes (non-dry-run):
   Define deleteFn, updateFn, createFn as closures that each call `breakers.Execute + resilience.WithRetry + sm.WithAccount` with the appropriate page object method (DeleteRecord, FillAndSubmit/UpdateRecord, FillAndSubmit/CreateRecord). Use op type labels "delete_record", "update_record", "create_record" for sm.WithAccount.

   Call: `results := reconcile.Apply(r.Context(), zoneID, plan, deleteFn, updateFn, createFn)`

9. Count errors and increment sync metrics:
   ```go
   hadErrors := false
   for _, res := range results {
       if res.Status == "error" {
           hadErrors = true
           if reg != nil {
               reg.SyncOpsTotal.WithLabelValues(res.Op, "error").Inc()
           }
       } else {
           if reg != nil {
               reg.SyncOpsTotal.WithLabelValues(res.Op, "ok").Inc()
           }
       }
   }
   ```

10. Write audit log (after apply — always, regardless of partial failure):
    ```go
    auditResult := "success"
    if hadErrors {
        auditResult = "failure"
    }
    if err := audit.Write(r.Context(), db, audit.Entry{
        TokenID:   claims.ID,
        AccountID: claims.AccountID,
        Action:    "sync",
        Resource:  "zone:" + zoneID,
        Result:    auditResult,
    }); err != nil {
        slog.ErrorContext(r.Context(), "audit log write failed", "error", err)
    }
    ```

11. Respond with full SyncResponse:
    Build a custom response struct (inline or add HadErrors to SyncResponse):
    ```go
    type syncHTTPResponse struct {
        DryRun    bool                  `json:"dry_run"`
        Plan      reconcile.SyncPlan    `json:"plan"`
        Results   []reconcile.SyncResult `json:"results"`
        HadErrors bool                  `json:"had_errors"`
    }
    response.WriteJSON(w, http.StatusOK, syncHTTPResponse{
        DryRun:    false,
        Plan:      plan,
        Results:   results,
        HadErrors: hadErrors,
    })
    ```

CRITICAL: Use HTTP 200 always (not 207) — HTTP 207 is uncommon in REST APIs and surprises clients; had_errors field in body signals partial failure.

CRITICAL: The `reconcile` package name is `reconcile` (not `sync`) — import path is `github.com/vnovakovic/dns-he-net-automation/internal/reconcile`.

Read pages package to find the correct constructor and method names for RecordPage (pages.NewRecordPage, ListRecords, DeleteRecord, CreateRecord/FillAndSubmit, UpdateRecord pattern).
