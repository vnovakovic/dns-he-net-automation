`RecordKey` struct:
```go
type RecordKey struct {
    Type     model.RecordType
    Name     string
    Content  string
    // SRV disambiguation fields (zero value for non-SRV)
    Priority int
    Weight   int
    Port     int
}
```

`keyOf(r model.Record) RecordKey` function:
- For SRV records (`r.Type == model.RecordTypeSRV`): include Priority, Weight, Port in key
- For all other types: Priority=0, Weight=0, Port=0 in key

`SyncPlan` struct with Add, Update, Delete slices of model.Record

`SyncResult` struct with Op string, Record model.Record, Status string ("ok"|"error"), ErrorMsg string

`SyncResponse` struct with DryRun bool, Plan SyncPlan, Results []SyncResult (json tags: dry_run, plan, results)

`DiffRecords(current, desired []model.Record) SyncPlan`:
- Build currentMap keyed by keyOf()
- Iterate desired: if not in map → Add; if in map but !recordsEqual() → Update (carry cur.ID into d); if identical → skip (mark matched)
- Remaining unmatched current records → Delete
- NOTE: matched keys tracking: for both "identical" and "update" cases, mark the key as matched so it is NOT added to Delete

`recordsEqual(a, b model.Record) bool`:
- Compare TTL, Priority, Weight, Port, Target, Dynamic only
- Do NOT compare Type/Name/Content (those are in the key already) or ID (IDs differ between current and desired)

`Apply(ctx context.Context, zoneID string, plan SyncPlan, deleteFn, updateFn, createFn func(context.Context, string, model.Record) error) []SyncResult`:
- Iterate plan.Delete → call deleteFn → collect opResult("delete", r, err)
- Iterate plan.Update → call updateFn → collect opResult("update", r, err)
- Iterate plan.Add → call createFn → collect opResult("add", r, err)
- NEVER return early on error — always process all items
- Return results (empty slice [], not nil, when plan is empty)

`opResult(op string, r model.Record, err error) SyncResult` helper.
